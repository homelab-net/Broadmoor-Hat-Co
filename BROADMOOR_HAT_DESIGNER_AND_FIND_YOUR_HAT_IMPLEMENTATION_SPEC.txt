# Broadmoor Hat Co — Hat Designer + Find Your Hat Implementation Specification

**Reference website:** https://broadmoorhatco.com/

## 1. Purpose

Build and deliver two new customer-facing pages for the Broadmoor Hat Co website:

1. **Hat Designer**
2. **Find Your Hat**

These pages must be implemented as part of a **complete Shopify theme codebase** that can be imported and deployed with minimal manual intervention. The output must include all baked-in logic, all required assets, and all required page behavior described in this document.

Do not build a detached microsite. Do not build a prototype. Do not build a partial component demo. Build a production-ready theme implementation package for the Broadmoor Hat Co website.

---

## 2. Core Product Intent

### 2.1 Hat Designer
The Hat Designer is a simple visual configurator for custom western hats.

Users must be able to change only the following:
- felt color
- brim
- crown
- band

The Hat Designer must visually update the assembled hat instantly when the user changes a selection.

### 2.2 Find Your Hat
Find Your Hat is a short quiz that helps a user identify a recommended western hat archetype.

The quiz must:
- ask 8 questions
- produce 1 of 4 results
- display a short explanation of the result
- map the result to a pre-baked baseline hat design
- allow the user to continue into the Hat Designer with that baseline design already loaded

## 5. Asset System and Compositing Contract

### 5.1 Canvas size
All piece-part images are normalized to:
- **1078 x 1078 pixels**

### 5.2 Alignment rule
All piece parts assemble by aligning from the **top-left pixel origin**.

This is a locked rule.
Do not compute dynamic anchor points.
Do not attempt auto-alignment.
Use fixed stack alignment.

### 5.3 Layer order
Render piece parts in the following order:
1. **brim**
2. **crown**
3. **band**

Brim is the bottom layer.
Band is the top layer.

### 5.4 Piece-part classes
The system uses three part classes:
- brim
- crown
- band

### 5.5 Naming convention
Use deterministic lowercase internal IDs in the format:
- `category_shape`

Examples:
- `crown_open`
- `brim_flat`
- `band_leather`

If the baked-in asset library uses specific filenames, internal logic must match those exact filenames.

### 5.6 UI labels
Dropdowns must derive from the asset library / piece-part filenames, but the user-facing UI must show **clean professional shorthand labels**.

Examples:
- internal: `brim_george`
- display: `George`

### 5.7 Existing modular production standard
The modular prompt standards already establish the photographic and compositing intent for crown, brim, and band assets. The crown-only template requires consistent crown seam placement and strict modular alignment fileciteturn0file1. The brim-only template requires a clean crown opening and exact compositing compatibility with the crown system fileciteturn0file0. The band-only modular prompt library requires a fitted crown-base overlay scaled for seamless drop-on compositing within the same production standard fileciteturn0file2.

## 7. Hat Designer Page Specification

### 7.1 Page name
**Hat Designer**

### 7.2 Purpose
Allow the user to visually configure a custom western hat by selecting:
- felt color
- brim
- crown
- band

### 7.3 Intro copy
Use this exact page intro or extremely close equivalent:

> Build your custom western hat by selecting your preferred brim, crown, band, and felt color. Your design updates instantly so you can compare styles, save your hat, and bring it into your appointment.

### 7.4 Default page load behavior
If the user enters the Hat Designer directly, load the **Western Formal** baseline design by default.

Default direct-load design:
- color: black
- crown: cattleman
- brim: George
- band: leather

### 7.5 Quiz-origin load behavior
If the user arrives from Find Your Hat, the page must load the baseline design associated with their quiz result.

Only in quiz-origin cases, display this message near the top of the page:

**Loaded from your Find Your Hat result: [RESULT_NAME]**

Do not show that message on direct page entry.

### 7.6 Approved page layout
Render the Hat Designer in this order:

1. page title
2. short intro text
3. assembled hat preview at the top
4. color circles
5. selected color text under the circles
6. brim dropdown
7. crown dropdown
8. band dropdown
9. action buttons

### 7.7 Dropdowns
Use three distinct dropdown controls:
- brim
- crown
- band

Use clean display labels in the UI.
Use internal asset IDs in logic/state.

### 7.8 Color selector UI
Render five selectable color circles corresponding to:
- pecan
- black
- chocolate
- silverbelly
- white

Use clear, premium-looking circles. Display the selected color text beneath them.

### 7.9 Preview behavior
The assembled hat preview must:
- be the primary focal element near the top of the page
- update instantly when any selection changes
- avoid visible loading states
- remain static with no rotations or alternate viewpoints

### 7.10 Action buttons on Hat Designer
The Hat Designer page must include these buttons:
- **Book Appointment**
- **Save PNG**
- **Copy Share Link**

Recommended order:
1. Book Appointment
2. Save PNG
3. Copy Share Link

### 7.11 Button behaviors
#### Book Appointment
Navigate the user to the existing Appointments page on the Broadmoor Hat Co website.

#### Save PNG
Export the current assembled design as a PNG according to the export specification in Section 10.

#### Copy Share Link
Copy a URL that reopens the current design with the exact currently selected:
- brim
- crown
- band
- color

## 9. Quiz Questions and Scoring

### 9.1 Archetype keys
Use these result keys:
- `western_formal`
- `everyday_classic`
- `bold_statement`
- `rugged_traditional`

### 9.2 Public result titles
Use these exact titles:
- **Western Formal**
- **Everyday Classic**
- **Bold Statement**
- **Rugged Traditional**

### 9.3 Scoring rule
Each answer contributes **1 point** to its matching archetype.

Highest total wins.

### 9.4 Tie-breaker logic
If there is a tie, resolve in this order:
1. Q1
2. Q7
3. Q8
4. default to **Everyday Classic**

### 9.5 Quiz questions

#### Q1. What are you most likely wearing this hat for?
- A. Weddings, formal events, or elevated occasions
- B. A little bit of everything
- C. Making a statement and standing out
- D. Everyday western wear, ranch, or heritage style

#### Q2. What kind of first impression do you want your hat to give?
- A. Polished and refined
- B. Timeless and easy to wear
- C. Confident and unforgettable
- D. Authentic and grounded

#### Q3. Which overall look feels most like you?
- A. Clean and elevated
- B. Classic and versatile
- C. Bold and fashion-forward
- D. Traditional and rugged

#### Q4. How much attention do you want your hat to draw?
- A. Subtle and tasteful
- B. Noticeable, but balanced
- C. A strong visual statement
- D. More functional than flashy

#### Q5. What best matches your wardrobe?
- A. Dress clothes, tailored looks, and formal westernwear
- B. Denim, boots, and everyday staples
- C. Standout outfits and elevated styling
- D. Workwear, heritage westernwear, and durable basics

#### Q6. Which shape direction appeals to you most?
- A. Sleek and refined
- B. Balanced and classic
- C. Sharp and striking
- D. Traditional and sturdy

#### Q7. What matters most in your custom hat?
- A. Looking elevated for important occasions
- B. Having one hat that works for almost everything
- C. Creating something unique
- D. Wearing something authentic and rooted in western tradition

#### Q8. Which outcome sounds best to you?
- A. A refined western hat for elevated moments
- B. A go-to western hat I can wear anywhere
- C. A standout custom hat with strong personality
- D. A traditional western hat with grit and character

### 9.6 Answer mapping
Use this answer-to-archetype mapping consistently across all questions:
- A = Western Formal
- B = Everyday Classic
- C = Bold Statement
- D = Rugged Traditional

## 11. Share State and URL Contract

### 11.1 Requirement
The current hat design must be shareable by URL.

### 11.2 Required shared state
The URL must preserve enough information to fully restore:
- selected brim
- selected crown
- selected band
- selected color
- quiz result origin only if relevant and available

### 11.3 Expected behavior
If a user opens a shared URL:
- the Hat Designer must restore that exact design state
- the preview must match the saved state
- the dropdowns and color selection must match the saved state

### 11.4 Implementation note
Use a stable, deterministic client-side encoding approach that is easy to maintain. Readability is preferred over unnecessary obfuscation.
