REQUIRED FEATURE BEHAVIOR

==================================================
A) HAT QUIZ
==================================================

Purpose:
A guided style-discovery flow for customers who do not know hat terminology.

Requirements:
- Accessible from the hamburger/menu as “Hat Quiz”
- Deterministic scoring only
- Same answers must produce the same primary result and alternates
- Output:
  - 1 primary recommendation
  - 2 alternate recommendations
  - short rationale/explanation
  - CTA/button to open the recommendation in Hat Designer with preloaded selections
- No pricing
- No lead time / turnaround
- No dead-end result state

Quiz UX goal:
“Help me figure out what style fits me.”

==================================================
B) HAT DESIGNER
==================================================

Purpose:
A modular live-preview configurator for users who want direct control.

Requirements:
- Accessible from the hamburger/menu as “Hat Designer”
- Use uploaded transparent PNG piece-part assets as the preview source of truth
- Modular live preview using composited layers
- Users must be able to change:
  - crown
  - brim
  - band (if included in the available asset/library scope)
  - color
- AI generated bands have not been created. Utilize plain color-shifted images that do not have a filter applied to them. Use simple ribbon/bars to create a band that uses several different colors. The color-changing filter should only apply to the brim and crown of the hat.
- Output/behavior:
  - live preview
  - build code
  - build summary
  - save/share actions
  - booking handoff action
  - compatibility messaging if certain combinations are blocked or adjusted
- No pricing
- No lead time / turnaround

Designer UX goal:
“Let me build and refine my hat visually.”

==================================================
C) COLOR SYSTEM
==================================================

This is a hard requirement.

The system must provide visible customer-facing color controls:
- color chips
- swatches

Preferred implementation:

- clear active state

Color rendering requirement:
- use static transparent PNG assets
- apply browser-side filter-based recoloring or equivalent front-end transform
- do NOT require separate PNG files for every colorway
- preserve texture/shading as much as possible
- selected color must persist in:
  - saved/share URLs
  - handoff payloads
  - resume state if implemented
